## 三大经典范式总结

### 1. ReAct（Reasoning + Acting）- 边想边做

**核心思想**：每一步都先推理（Thought），再行动（Action），然后根据观察结果（Observation）决定下一步。

**工作流程**：
```
Thought: 分析当前情况，决定下一步
Action: 执行工具调用
Observation: 接收结果反馈
→ 回到 Thought（进入下一轮）
```

**适用场景**：
- 需要根据中间结果动态调整策略
- 环境信息不完全，需要逐步探索
- 问题复杂度未知，难以提前规划

**类比**：边爬山边看路，遇到障碍就换方向

**实现文件**：`react_agent.py`

---

### 2. Plan-and-Solve - 先谋后动

**核心思想**：分为两个明确阶段：先制定完整计划（Plan），再按计划执行（Solve）。

**工作流程**：
```
【阶段1：制定计划】
Plan:
  Step 1: ...
  Step 2: ...
  Step 3: ...

【阶段2：执行计划】
逐步执行每个步骤，收集结果

【生成最终答案】
综合所有结果
```

**适用场景**：
- 任务目标明确，步骤可预见
- 需要一次性完成多个子任务
- 追求执行效率和逻辑清晰性

**类比**：看完地图规划好路线，然后按路线走

**实现文件**：`plan_solve_agent.py`

---

### 3. Reflection - 执行后反思

**核心思想**：形成"执行 → 自我评估 → 发现问题 → 改进"的迭代循环。

**工作流程**：
```
【迭代循环】
Execute: 执行任务，生成输出
  ↓
Reflect: 自我评估
  - 优点：...
  - 问题：...
  - 改进建议：...
  - 评分：X/10
  ↓
Decision: 
  - 如果评分 >= 8 → 完成
  - 如果评分 < 8 → 根据建议改进，回到 Execute
```

**适用场景**：
- 对输出质量要求极高
- 任务允许多次迭代优化
- 需要自我纠错能力

**类比**：写完文章后反复修改打磨

**实现文件**：`reflection_agent.py`

## 核心洞察

这三个范式本质上都在回答同一个问题：

> **"怎么把 LLM 的一次性输出，升级为能完成复杂任务的持续循环？"**

- **ReAct** 的答案：让 LLM 每步都能看到上一步结果，动态调整
- **Plan-and-Solve** 的答案：让 LLM 先全局规划，再分步执行
- **Reflection** 的答案：让 LLM 评估自己的输出，发现问题就重来

## 如何选择范式？

### 决策流程

1. **质量要求极高？**
   - 是 → 用 **Reflection**

2. **任务步骤明确可预见？**
   - 是 → 用 **Plan-and-Solve**
   - 否 → 继续判断

3. **需要根据中间结果动态调整？**
   - 是 → 用 **ReAct**（默认选择）

### 实战建议

- **默认用 ReAct**：最通用、最灵活
- **明确流程用 Plan-and-Solve**：旅行规划、数据处理
- **高质量输出用 Reflection**：代码生成、文章撰写
- **可以混合使用**：先探索（ReAct）→ 再执行（Plan-and-Solve）→ 最后打磨（Reflection）
