## 12.2 BFCL：工具调用能力评估

### 12.2.1 BFCL 基准介绍

**BFCL = Berkeley Function-Calling Leaderboard**

**由加州大学伯克利分校推出，专门评估：**
- LLM能否正确理解工具描述
- LLM能否选择正确的工具
- LLM能否生成正确的参数

**评估维度：**

```
1. Simple (简单场景)
   - 单个工具调用
   - 参数简单

2. Multiple (多工具场景)
   - 需要调用多个工具
   - 有先后顺序

3. Parallel (并行场景)
   - 同时调用多个工具
   - 无依赖关系

4. Relevance (相关性判断)
   - 判断是否需要调用工具
   - 避免过度调用
```

**示例测试用例：**

```json
{
  "id": "test_001",
  "question": "查询北京今天的天气，并将结果翻译成英文",
  "available_tools": [
    {
      "name": "get_weather",
      "description": "获取指定城市的天气信息",
      "parameters": {
        "city": {"type": "string", "description": "城市名称"}
      }
    },
    {
      "name": "translate",
      "description": "翻译文本",
      "parameters": {
        "text": {"type": "string"},
        "target_lang": {"type": "string"}
      }
    }
  ],
  "expected_calls": [
    {
      "tool": "get_weather",
      "arguments": {"city": "北京"}
    },
    {
      "tool": "translate",
      "arguments": {
        "text": "<weather_result>",
        "target_lang": "English"
      }
    }
  ]
}
```

### 12.2.2 在 HelloAgents 中实现 BFCL 评估

**评估流程：**

```python
from hello_agents import SimpleAgent, HelloAgentsLLM
from hello_agents.evaluation import BFCLEvaluator

# 1. 加载BFCL数据集
evaluator = BFCLEvaluator()
test_cases = evaluator.load_dataset()

# 2. 创建被测智能体
agent = SimpleAgent(
    name="测试智能体",
    llm=HelloAgentsLLM(model="gpt-4"),
    tools=evaluator.get_tools()
)

# 3. 运行评估
results = []
for test_case in test_cases:
    # 智能体处理
    response = agent.run(test_case["question"])
    
    # 提取工具调用
    actual_calls = extract_tool_calls(response)
    expected_calls = test_case["expected_calls"]
    
    # 评分
    score = evaluator.score(actual_calls, expected_calls)
    results.append(score)

# 4. 生成报告
report = evaluator.generate_report(results)
print(report)
```

**评分标准：**

```python
def score_tool_call(actual, expected):
    """评估单个工具调用"""
    score = 0.0
    
    # 1. 工具名称正确 (40%)
    if actual["tool"] == expected["tool"]:
        score += 0.4
    
    # 2. 参数名称正确 (30%)
    if set(actual["arguments"].keys()) == set(expected["arguments"].keys()):
        score += 0.3
    
    # 3. 参数值正确 (30%)
    correct_values = 0
    for key in expected["arguments"]:
        if key in actual["arguments"]:
            if actual["arguments"][key] == expected["arguments"][key]:
                correct_values += 1
    
    if len(expected["arguments"]) > 0:
        score += 0.3 * (correct_values / len(expected["arguments"]))
    
    return score
```

```

- 加强多步工具调用训练

```

## 12.3 GAIA：通用 AI 助手能力评估

### 12.3.1 GAIA 基准介绍

**GAIA = General AI Assistants**

**特点：**
- 由Meta AI Research和HuggingFace联合推出
- 被认为是"最难的AI评估基准"
- 问题需要真实的推理和工具使用

**难度分级：**

```
Level 1 (简单): 
- 单步推理
- 基础工具使用
- 示例："查询埃菲尔铁塔的高度"

Level 2 (中等):
- 多步推理
- 组合多个工具
- 示例："找出2020年奥斯卡最佳影片的导演，并查询他的年龄"

Level 3 (困难):
- 复杂推理链
- 需要创造性解决问题
- 示例："分析这个Excel文件，找出销售额最高的季度，并预测下一季度趋势"
```

**示例题目（Level 3）：**

```
问题：
附件中是一个CSV文件，包含某公司2023年的销售数据。
请分析：
1. 哪个月份的销售额最高？
2. 环比增长率最大的是哪个月？
3. 如果保持这个增长趋势，预计2024年Q1的总销售额是多少？

提供的文件：sales_2023.csv

期望答案格式：
1. X月 (XX万元)
2. Y月 (增长率ZZ%)
3. 预计QQ万元
```

```python
    try:
        # 智能体回答
        answer = agent.run(
            question["question"],
            files=question.get("files", [])

            answer=answer,
            ground_truth=question["answer"],
            question_id=question["id"]
        )

            "id": question["id"],
            "correct": is_correct,
            "answer": answer
        })

        results.append({
            "id": question["id"],
            "correct": False,
            "error": str(e)
        })

# 4. 计算准确率
accuracy = sum(r["correct"] for r in results) / len(results)
print(f"GAIA Level 2 准确率: {accuracy * 100:.1f}%")
```

## 12.4 数据生成质量评估

### 12.4.1 评估方法概述

**场景：** 你用智能体生成了1000道数学题，如何评估质量？

**三种评估方法：**

1. **人工评估** - 最准确，但成本高
2. **LLM Judge** - 用强大的LLM评审
3. **Win Rate** - 两两对比，计算胜率

### 12.4.2 LLM Judge 评估工具

**原理：** 让GPT-4/Claude等强大模型充当"评审员"

```python
class LLMJudge:
    def __init__(self, judge_model="gpt-4"):
        self.llm = HelloAgentsLLM(model=judge_model)
    
    def evaluate(self, question, answer, criteria):
        """评估单个答案"""
        
        prompt = f"""
        你是一位严格的评审专家。请评估以下答案的质量。
        
        问题：{question}
        答案：{answer}
        
        评估维度：
        {self._format_criteria(criteria)}
        
        请对每个维度打分（0-10分），并给出总评。
        
        输出JSON格式：
        {{
          "scores": {{"维度1": 分数, "维度2": 分数, ...}},
          "total": 总分,
          "comments": "评语"
        }}
        """
        
        result = self.llm.generate(prompt)
        return json.loads(result)
    
    def batch_evaluate(self, qa_pairs):
        """批量评估"""
        results = []
        for qa in qa_pairs:
            score = self.evaluate(
                question=qa["question"],
                answer=qa["answer"],
                criteria=["正确性", "清晰度", "完整性"]
            )
            results.append(score)
        return results

# 使用示例
judge = LLMJudge(judge_model="gpt-4")

results = judge.batch_evaluate([
    {
        "question": "2+2等于几？",
        "answer": "2+2=4"
    },
    {
        "question": "解释量子纠缠",
        "answer": "量子纠缠是指..."
    }
])

print(f"平均得分: {np.mean([r['total'] for r in results])}")
```

### 12.4.3 Win Rate 评估工具

**原理：** A vs B，让评审员判断谁更好

```python
class WinRateEvaluator:
    def __init__(self, judge_llm):
        self.judge = judge_llm
    
    def compare(self, question, answer_a, answer_b):
        """对比两个答案"""
        
        prompt = f"""
        问题：{question}
        
        答案A：{answer_a}
        
        答案B：{answer_b}
        
        请判断哪个答案更好，或者打平。
        
        输出格式：
        - "A" 表示答案A更好
        - "B" 表示答案B更好
        - "Tie" 表示打平
        
        只输出一个字母，不要解释。
        """
        
        result = self.judge.generate(prompt).strip()
        return result
    
    def calculate_win_rate(self, model_a_answers, model_b_answers, questions):
        """计算模型A对模型B的胜率"""
        
        wins = 0
        losses = 0
        ties = 0
        
        for q, ans_a, ans_b in zip(questions, model_a_answers, model_b_answers):
            result = self.compare(q, ans_a, ans_b)
            
            if result == "A":
                wins += 1
            elif result == "B":
                losses += 1
            else:
                ties += 1
        
        total = wins + losses + ties
        win_rate = (wins + 0.5 * ties) / total
        
        return {
            "win_rate": win_rate,
            "wins": wins,
            "losses": losses,
            "ties": ties
        }

# 使用示例
evaluator = WinRateEvaluator(judge_llm=gpt4)

result = evaluator.calculate_win_rate(
    model_a_answers=my_model_outputs,
    model_b_answers=gpt4_outputs,
    questions=test_questions
)

print(f"胜率: {result['win_rate'] * 100:.1f}%")
print(f"胜/平/负: {result['wins']}/{result['ties']}/{result['losses']}")
```

## 12.5 本章总结

### 核心要点

1. **评估是智能体开发的必要环节**
   - 量化能力水平
   - 发现改进方向
   - 与业界对比

2. **选择合适的评估基准**
   - BFCL：工具调用能力
   - GAIA：通用助手能力（最难）
   - 根据应用场景选择

3. **评估方法多样**
   - 自动评估：快速、可重复
   - LLM Judge：灵活、接近人类判断
   - Win Rate：相对比较、直观

### 最佳实践

- ✅ 建立持续评估流程（CI/CD）
- ✅ 使用多个基准全面评估
- ✅ 记录每次评估结果追踪进步
- ✅ 分析错误案例针对性改进
- ✅ 对比业界SOTA模型找差距
