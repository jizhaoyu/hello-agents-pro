## 9.1 什么是上下文工程

**定义：**
上下文工程是指为智能体精心设计、构建和优化输入上下文的过程，确保在有限的上下文窗口内提供最有价值的信息。

**核心挑战：**

1. **上下文窗口限制** - 即使GPT-4有128K tokens，实际可用空间仍有限
2. **信息密度** - 并非所有信息都同等重要
3. **成本控制** - 更长的上下文意味着更高的API成本
4. **推理质量** - 过多噪音信息会降低推理准确度

**类比理解：**
```
上下文工程 ≈ 图书馆管理员

你去图书馆问："有关人工智能的书吗？"

糟糕的管理员：
"我们有10000本书，全部都在这里，你自己找吧。"
（信息过载，无法处理）

优秀的管理员：
"AI相关的书在B区3层，我推荐这3本最经典的：
1. 《智能时代》- 概述性著作
2. 《深度学习》- 技术细节
3. 《AI伦理》- 社会影响
需要更具体的推荐吗？"
（精准检索，结构化组织）
```

## 9.2 为什么上下文工程重要

### 9.2.1 有效上下文的"解剖学"

**一个有效的上下文应包含：**

```
┌────────────────────────────────────┐
│      系统提示 (System Prompt)        │ ← 角色定位、行为准则
├────────────────────────────────────┤
│      任务上下文 (Task Context)       │ ← 当前任务的目标和要求
├────────────────────────────────────┤
│    相关知识 (Relevant Knowledge)     │ ← 从知识库检索的信息
├────────────────────────────────────┤
│    历史记忆 (Memory)                 │ ← 过往交互的关键信息
├────────────────────────────────────┤
│    工具列表 (Available Tools)        │ ← 可用工具及其使用说明
├────────────────────────────────────┤
│    约束条件 (Constraints)            │ ← 限制、注意事项
├────────────────────────────────────┤
│    示例 (Examples)                   │ ← Few-shot学习样本
├────────────────────────────────────┤
│    用户输入 (User Input)             │ ← 当前问题
└────────────────────────────────────┘
```

**反例 - 糟糕的上下文：**
```python
prompt = f"""
你是一个AI助手。

这是我们的代码库，共有1000个文件：
{dump_all_code()}  # 100K+ tokens!

用户问题：{user_question}
"""
```

**正例 - 优化的上下文：**
```python
# 1. 检索相关文件
relevant_files = search_codebase(user_question, top_k=3)

# 2. 提取关键信息
summaries = [summarize(f) for f in relevant_files]

# 3. 构建精简上下文
prompt = f"""
你是一个代码审查助手。

任务：{user_question}

相关代码文件（共3个）：
{format_code_snippets(relevant_files)}

关键信息：
{join(summaries)}

请基于以上信息回答。
"""
```

### 9.2.2 上下文检索与智能体式搜索

传统的文档检索是"一次性"的，而智能体式搜索是"迭代"的：

```python
# 传统检索
docs = vector_db.search(query, top_k=5)
answer = llm.generate(query, context=docs)

# 智能体式搜索（上下文工程）
def agent_search(query):
    context = ContextBuilder()
    
    # 第1轮：初步检索
    initial_docs = vector_db.search(query, top_k=10)
    context.add(initial_docs)
    
    # 第2轮：分析缺失信息
    gaps = llm.analyze_gaps(query, context)
    
    # 第3轮：补充检索
    for gap in gaps:
        additional_docs = vector_db.search(gap, top_k=3)
        context.add(additional_docs)
    
    # 第4轮：压缩与结构化
    context.compress()
    context.structure()
    
    return llm.generate(query, context=context.build())
```

### 9.2.3 面向长时程任务的上下文工程

**长时程任务的特点：**
- 需要多步推理
- 中间状态复杂
- 上下文不断演化
- 容易偏离目标

**解决方案：动态上下文管理**

```python
class LongTermTask:
    def __init__(self):
        self.context = ContextBuilder()
        self.note_tool = NoteTool()  # 持久化笔记
        self.progress = []
    
    def execute(self, goal):
        # 初始化上下文
        self.context.add_goal(goal)
        
        while not self.is_complete():
            # 1. 构建当前步骤的上下文
            current_context = self.build_step_context()
            
            # 2. 执行推理
            action = self.agent.decide(current_context)
            result = self.execute_action(action)
            
            # 3. 更新笔记（压缩历史）
            self.note_tool.append(f"""
            步骤 {len(self.progress)}:
            行动: {action}
            结果: {result}
            """)
            
            # 4. 清理上下文（避免溢出）
            self.context.clear_old_steps()
            
            # 5. 记录进度
            self.progress.append(result)
    
    def build_step_context(self):
        """构建当前步骤的精简上下文"""
        return {
            "goal": self.context.goal,
            "recent_steps": self.progress[-3:],  # 只保留最近3步
            "summary": self.note_tool.get_summary(),  # 历史总结
            "current_state": self.get_current_state()
        }
```

## 9.3 在 Hello-Agents 中的实践：ContextBuilder

### 9.3.1 GSSC 流水线详解

**GSSC = Gather → Select → Structure → Compress**

```python
class ContextBuilder:
    def __init__(self, max_tokens=8000):
        self.max_tokens = max_tokens
        self.items = []
    
    # G - Gather (收集)
    def gather(self, *sources):
        """从多个来源收集信息"""
        for source in sources:
            if isinstance(source, str):
                self.items.append({"type": "text", "content": source})
            elif isinstance(source, dict):
                self.items.append(source)
    
    # S - Select (筛选)
    def select(self, query, top_k=5):
        """基于相关性筛选最重要的信息"""
        # 计算每个item与query的相关性
        scored_items = []
        for item in self.items:
            score = self.relevance_score(query, item)
            scored_items.append((score, item))
        
        # 排序并取Top-K
        scored_items.sort(reverse=True, key=lambda x: x[0])
        self.items = [item for score, item in scored_items[:top_k]]
    
    # S - Structure (结构化)
    def structure(self):
        """将信息组织成结构化格式"""
        structured = {
            "background": [],
            "facts": [],
            "code": [],
            "examples": []
        }
        
        for item in self.items:
            item_type = self.classify_item(item)
            structured[item_type].append(item)
        
        self.items = structured
    
    # C - Compress (压缩)
    def compress(self):
        """压缩信息以适应token限制"""
        current_tokens = self.count_tokens()
        
        if current_tokens > self.max_tokens:
            # 策略1: 移除低相关性内容
            self.remove_low_relevance()
            
            # 策略2: 总结长文本
            self.summarize_long_texts()
            
            # 策略3: 提取关键信息
            self.extract_key_points()
    
    def build(self):
        """构建最终的上下文字符串"""
        return self.format_context(self.items)
```

**使用示例：**

```python
# 构建上下文
context = ContextBuilder(max_tokens=8000)

# G - 收集信息
context.gather(
    code_files,           # 代码文件
    documentation,        # 文档
    issue_description,    # 问题描述
    related_commits       # 相关提交
)

# S - 筛选相关信息
context.select(query="修复登录bug", top_k=5)

# S - 结构化组织
context.structure()

# C - 压缩到限制内
context.compress()

# 生成最终上下文
final_context = context.build()
```

### 9.4.2 核心操作详解

```python
class NoteTool:
    def __init__(self, filepath=None):
        self.sections = []
        self.filepath = filepath
    
    def create_section(self, title):
        """创建新章节"""
        self.sections.append({
            "title": title,
            "content": [],
            "timestamp": datetime.now()
        })
    
    def append(self, content):
        """在当前章节添加内容"""
        if not self.sections:
            self.create_section("默认章节")
        
        self.sections[-1]["content"].append({
            "text": content,
            "timestamp": datetime.now()
        })
    
    def search(self, keyword):
        """搜索笔记内容"""
        results = []
        for section in self.sections:
            if keyword in section["title"]:
                results.append(section)
            for item in section["content"]:
                if keyword in item["text"]:
                    results.append(item)
        return results
    
    def summarize(self, llm=None):
        """总结笔记内容"""
        full_text = self.export()
        
        if llm:
            prompt = f"""
            请总结以下笔记的核心内容：
            
            {full_text}
            
            要求：
            1. 提炼关键要点（不超过5条）
            2. 保留重要的结论和发现
            3. 忽略过程细节
            """
            return llm.generate(prompt)
        else:
            # 简单版本：提取每个章节的标题
            return "\n".join([s["title"] for s in self.sections])
    
    def export(self, format="markdown"):
        """导出笔记"""
        if format == "markdown":
            output = []
            for section in self.sections:
                output.append(f"## {section['title']}\n")
                for item in section["content"]:
                    output.append(f"- {item['text']}\n")
                output.append("\n")
            return "".join(output)
```

## 9.5 TerminalTool：即时文件系统访问

### 9.5.1 设计理念与安全机制

**为什么需要 TerminalTool？**

在代码库维护等任务中，智能体需要：
- 查看文件内容
- 搜索特定模式
- 列出目录结构
- 执行简单脚本

**安全机制：**

```python
class TerminalTool:
    def __init__(self, workspace_root, allowed_commands=None):
        self.workspace_root = Path(workspace_root).resolve()
        
        # 白名单命令
        self.allowed_commands = allowed_commands or [
            "ls", "cat", "grep", "find", "tree"
        ]
        
        # 禁止的危险命令
        self.forbidden_commands = [
            "rm", "del", "format", "dd", 
            "sudo", "chmod", "chown"
        ]
    
    def execute(self, command):
        """安全执行命令"""
        # 1. 解析命令
        cmd_parts = command.split()
        base_cmd = cmd_parts[0]
        
        # 2. 安全检查
        if base_cmd in self.forbidden_commands:
            raise SecurityError(f"禁止执行危险命令: {base_cmd}")
        
        if base_cmd not in self.allowed_commands:
            raise SecurityError(f"命令不在白名单中: {base_cmd}")
        
        # 3. 路径检查（防止目录遍历）
        for part in cmd_parts:
            if self.is_path(part):
                abs_path = (self.workspace_root / part).resolve()
                if not abs_path.is_relative_to(self.workspace_root):
                    raise SecurityError("不允许访问工作区外的路径")
        
        # 4. 执行命令
        try:
            result = subprocess.run(
                command,
                shell=True,
                capture_output=True,
                text=True,
                timeout=10,
                cwd=self.workspace_root
            )
            return result.stdout
        except subprocess.TimeoutExpired:
            return "命令执行超时"
```

### 9.5.2 核心功能详解

```python
# 使用示例
terminal = TerminalTool(workspace_root="/project")

# 查看文件
content = terminal.read_file("src/main.py")

# 搜索模式
results = terminal.grep("TODO", recursive=True)

# 列出目录
files = terminal.list_dir("src/", pattern="*.py")

# 查找文件
found = terminal.find("config.yaml")
```

## 9.6 长程智能体实战：代码库维护助手

### 完整示例

```python
class CodebaseMaintenanceAgent:
    def __init__(self, repo_path):
        self.repo_path = repo_path
        self.terminal = TerminalTool(repo_path)
        self.note = NoteTool()
        self.context = ContextBuilder(max_tokens=8000)
        self.llm = HelloAgentsLLM()
    
    def analyze_issue(self, issue_description):
        """分析并修复代码库问题"""
        
        # 步骤1: 记录任务
        self.note.create_section("任务")
        self.note.append(f"问题描述: {issue_description}")
        
        # 步骤2: 搜索相关文件
        keywords = self.extract_keywords(issue_description)
        relevant_files = []
        
        for keyword in keywords:
            results = self.terminal.grep(keyword, recursive=True)
            relevant_files.extend(results)
        
        self.note.create_section("相关文件")
        self.note.append(f"找到 {len(relevant_files)} 个相关文件")
        
        # 步骤3: 构建上下文
        self.context.gather(
            issue_description,
            relevant_files[:5]  # 只取前5个最相关的
        )
        self.context.select(issue_description, top_k=3)
        self.context.compress()
        
        # 步骤4: 分析问题
        analysis_prompt = f"""
        基于以下信息分析问题：
        
        {self.context.build()}
        
        请分析：
        1. 问题的根本原因
        2. 涉及的代码模块
        3. 建议的修复方案
        """
        
        analysis = self.llm.generate(analysis_prompt)
        
        self.note.create_section("问题分析")
        self.note.append(analysis)
        
        # 步骤5: 生成修复代码
        fix_prompt = f"""
        基于以下分析生成修复代码：
        
        {analysis}
        
        原始代码：
        {self.context.get_code_snippets()}
        
        请提供具体的代码修复方案。
        """
        
        fix = self.llm.generate(fix_prompt)
        
        self.note.create_section("修复方案")
        self.note.append(fix)
        
        # 返回完整报告
        return {
            "analysis": analysis,
            "fix": fix,
            "note": self.note.export()
        }

# 使用
agent = CodebaseMaintenanceAgent("/path/to/repo")
result = agent.analyze_issue("用户登录后session立即过期")

print(result["note"])
```

## 9.7 本章总结

### 核心要点

1. **上下文工程是智能体开发的关键技术**
   - 有限的上下文窗口需要精心设计
   - 信息密度比信息量更重要
   - 动态管理优于静态拼接

2. **GSSC流水线提供系统化方法**
   - Gather: 从多源收集信息
   - Select: 基于相关性筛选
   - Structure: 组织成结构化格式
   - Compress: 压缩到token限制内

3. **工具协同提升长时程能力**
   - NoteTool: 持久化中间结果
   - TerminalTool: 即时文件访问
   - ContextBuilder: 动态上下文构建

### 最佳实践

- ✅ 始终监控上下文token使用量
- ✅ 优先保留最相关的信息
- ✅ 使用结构化格式组织上下文
- ✅ 为长任务维护压缩的历史记录
- ✅ 实施严格的安全控制
