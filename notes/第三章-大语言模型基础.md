## 3.1 语言模型与 Transformer 架构

### 3.1.1 从 N-gram 到 RNN:三次"不够用"

这条演进线是用第二章的"问题驱动"模式来讲述的。每一步解法都在填上一个坑,但自己又挖了一个新坑。

#### 📊 第一代:N-gram(统计语言模型)

**核心**:一个词的出现概率,只看它前面 N-1 个词(马尔可夫假设)。Bigram(N=2)只看前 1 个,Trigram(N=3)只看前 2 个。

**数学**:
```
P(句子) = P(w1) × P(w2|w1) × P(w3|w2) × … × P(wn|wn-1)    ← Bigram 版
```

这些条件概率怎么来的?**数出来的。** 在语料库里数"w1 后面跟 w2 出现了多少次"除以"w1 总共出现了多少次"。

**类比**:一个只认得"上一道菜"的美食评论家——"这道牛排之后,通常会上甜品还是咖啡?我翻翻我的小本本…牛排后面 80%是甜品,20%是咖啡。"

**败在**:
- **数据稀疏**:语料库里没见过的词对 → 概率直接判 0,彻底堵死。虽然可以用"平滑"技术打补丁,但根子上的问题没解决。
- **无语义**:`agent` 和 `robot` 在它眼里是两个完全无关的符号——它无法理解这俩词"意思接近"。

---

#### 🧬 第二代:神经网络语言模型 + 词嵌入(Bengio,2003)

**核心**:不再把词当离散符号,而是用一个**连续的高维向量**来表示每个词。语义相近的词,向量在空间中离得近。

**类比**:把每个词想象成地图上的一个坐标。"国王"在(0.9,0.8),"王后"在(0.9,0.2),"男人"在(0.7,0.9),"女人"在(0.7,0.3)。

```
king - man + woman = (0.9,0.8) - (0.7,0.9) + (0.7,0.3) = (0.9,0.2) ≈ queen!
```

> 🎯 它在向量空间里做了一道"语义平移":从"国王"出发,减去"男性"的方向,加上"女性"的方向,就自动到达了"女王"。**这说明词向量不是死记硬背,而是真的学到了一种抽象的"性别"维度。**

**填了 N-gram 的"无语义"坑,但自己埋了新坑:上下文窗口仍然是固定的**——只能看前 N-1 个词,窗口之外的历史就当没发生过。

---

#### 🔁 第三代:RNN / LSTM

**核心**:引入**隐藏状态(Hidden State)**——网络自己的"短期记忆"。每一步:读当前输入 + 回忆上一刻的"记了什么" → 产生新的记忆 → 传给下一刻。

**类比**:RNN 像传话游戏。第一个人听到"今天天气真好",小声告诉第二个人;第二个人结合自己听到的+记忆告诉第三个人…LSTM 则像是传话时允许你"选择性遗忘不重要的话"——加了三扇门来控制哪些信息该保留、哪些该丢掉。

**LSTM 的三扇门**(这是工程上的精妙设计):

| 门 | 作用 | 类比 |
|----|------|------|
| **遗忘门** Forget Gate | 决定从上一刻的状态中**扔掉**哪些信息 | "前面说的那个细节不重要的,忘了吧" |
| **输入门** Input Gate | 决定当前输入中哪些新信息值得**存进去** | "这个信息很关键,记下来" |
| **输出门** Output Gate | 决定根据当前状态,**输出**哪些信息给隐藏状态 | "现在你该知道的是这些…" |

**填了"固定窗口"的坑,但埋了一个要命的坑:必须串行。** 第 t 步必须等第 t-1 步算完才知道自己的输入是什么。这导致两个后果:① 无法并行→训练慢到令人发指;② 序列很长时,梯度在反向传播中连乘→趋近于 0(梯度消失),靠前的词对最终结果几乎没影响。

### 🔥 3.1.2 Transformer:把"必须按顺序"这个前提本身给干掉

2017 年,Google 团队问了一句:"**如果我不按时间步处理呢?如果每个词可以同时、直接地看全句所有词呢?**"

答案是:你要的不是循环,是**注意力(Attention)**。

#### 自注意力 Self-Attention:全章最重要的概念

> **每个词都要"向全句所有词打听一遍",搞清楚自己在这个语境里该表示什么。**

为每个词元引入三个**可学习的角色**:

| 角色 | 全称 | 含义 | 🎓 开卷考试类比 |
|------|------|------|---------------|
| **Q** | Query 查询 | 当前词"想问什么"——"我在找什么信息?" | 你手上那道**考题** |
| **K** | Key 键 | 每个词的"标签"——"我身上有什么信息可查?" | 每本参考书的**目录** |
| **V** | Value 值 | 每个词携带的"实际内容"——"我真正装了什么?" | 每本参考书的**正文** |

计算过程四步:

```
① Q 和所有 K 做点积 →   "这道考题(我)跟哪本书的目录最匹配?"  → 得到每个词的相关性得分
② 除以 √d_k        →   "得分别太悬殊,帮 Softmax 正常发挥"  → 缩放,防梯度过小
③ Softmax          →   "把得分变成百分比权重,总和=100%"     → 归一化
④ 权重 × V 再加总   →   "按权重从各本书里取内容,汇总成答案"   → 得到融合全局上下文的新表示
```

**一条公式**:
$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) \times V$$

> 🎯 原文例子:**"The agent learns because **it** is intelligent."** 当模型处理加粗的 "it" 时,自注意力会自动给 "agent" 分配最高权重——模型知道了:"哦,it 指的是 agent。"

---

#### 多头注意力 Multi-Head Attention:不只请一个专家

单头注意力可能每次只抓到一种关系(比如指代)。但语言里的关系是立体的:指代、时态、修饰、因果、情感…

**解法**:把 Q/K/V 在维度上切成 h 份(h="头数"),**每一份独立做一次注意力计算** → 最后把 h 个"专家"的意见拼接起来,再过一个线性变换整合。

| | 类比 |
|---|------|
| **单头** | 请一个专家看 X 光片——他可能只看得懂骨折,看不出肿瘤 |
| **多头** | 请 h 个不同专科的专家同时看——骨科看骨头、肿瘤科看阴影、放射科看密度 → 最后汇总,诊断更全面 |

---

#### 其余三个关键组件

| 组件 | 干什么 | 类比 | 为什么不能没有 |
|------|--------|------|--------------|
| **前馈网络 FFN** | 每个位置独立过:Linear(d→4d) → ReLU → Linear(4d→d) | 注意力像"开会讨论"(各人交换意见)→FFN 像"回去自己消化整理笔记" | 注意力只负责**聚合信息**,FFN 负责从聚合后的信息里**提取更高阶的特征** |
| **残差连接 + 层归一化** | 输入 x **原样加到**子层输出上("残差"),再归一化 | 残差 = **高速公路**:梯度可以绕过子层直传,不怕网络深。归一化 = **保持队形**:每层的输入分布都整齐划一,训练又快又稳 | 没有残差→深层网络训练不动;没有归一化→每一层的输入分布都在"漂移",训练像追移动靶 |
| **位置编码** | 用 sin/cos 给每个位置算唯一向量,加到词嵌入上 | 给每个词穿一件**印着座位号的背心**。注意力本身分不清顺序,"agent learns"和"learns agent"对它是同一盘菜 | 注意力是全连接的、无顺序的——位置编码是唯一告诉模型"谁是第几个"的信号 |

> 🔥 **拼图拼全了**:**自注意力 = 全局关联捕获 | 多头 = 多角度观察 | FFN = 深加工 | 残差 = 不丢梯度 | 位置编码 = 不丢顺序。** 五个组件各司其职,才组成了这个可以大规模并行训练的发动机。

### 3.1.3 Decoder-Only:GPT 把编码器整条砍掉了

原版 Transformer 有编码器(Encoder,负责"理解输入")+ 解码器(Decoder,负责"生成输出")。

GPT 的开发团队问了一个很"暴力"的问题:

> **语言的核心任务,不就是预测下一个最有可能出现的词吗?** 回答问题?→在问题后面续写答案。写代码?→在注释后面续写代码。写故事?→在开头后面续写结尾。**全都是"文字接龙"。**

于是他们做了减法:**编码器,整条砍掉,只留解码器。** 这就叫 **Decoder-Only 架构**。

#### 自回归 Autoregressive:吃自己吐出来的东西

工作模式极其简单:
```
1. 给你一个开头 "Datawhale Agent is"
2. 模型预测下一个词 "a"
3. 把 "a" 接到末尾 → 新输入 = "Datawhale Agent is a"
4. 模型预测下一个词 "powerful"
5. 把 "powerful" 接上去 → 新输入 = "Datawhale Agent is a powerful"
6. …一直重复,直到够了或该停了
```

**类比**:一条贪吃蛇,每吃一个 token 身体就长一节,下一口饭的落点取决于它现在的整条身长。

#### 掩码自注意力 Masked Self-Attention:不许偷看未来

Decoder-Only 有个关键约束:**预测第 t 个词时,不允许偷看第 t+1 个及以后的东西。** 不然就成作弊了。

> 做法:在自注意力分数矩阵里,把所有"当前位置之后"的位置的分数设为 **-∞**(负无穷)。经过 Softmax 后,-∞ 变成概率 0——**数学上堵死**。

**类比**:考试时,你只能看到已经答过的题和正在答的这道题,后面的题被一块黑布遮住了。

#### 为什么现在几乎所有大模型都是 Decoder-Only?

| 原因 | 解释 |
|------|------|
| **训练目标统一** | 全宇宙只有一个任务:预测下一个词。这个任务可以在任何无标注文本上跑,不需要人工标注 |
| **结构简单,易于扩展** | 组件少了,工程瓶颈也少了。从 GPT-1 的 1.17 亿参数到 GPT-4 的数万亿参数,**骨架没变** |
| **天然适合生成任务** | 对话/写代码/翻译/总结…全是"续写"的不同变体,一个范式全搞定 |

## 3.2 与 LLM 的交互

### 3.2.1 提示工程:你怎么"驾驶"这台引擎

如果说 LLM 是一台引擎,那 Prompt 就是方向盘+油门+刹车。你说什么、怎么说,直接决定它往哪开、开多快。

#### 🌡️ 采样三旋钮:控制"下一个词选什么"

模型每步都面临一个**概率菜单**:几千个候选词,每个都有自己的"当选概率"。采样参数就是**你怎么从这份菜单上点菜**。

**Temperature 温度——"你敢不敢冒险?"**

$$P(token_i) = \text{softmax}\left(\frac{\text{logit}_i}{T}\right)$$

| T 值 | 概率分布变形为 | 效果 | 🍽️ 点菜类比 |
|------|--------------|------|------------|
| **低**(0~0.3) | 高概率项被放得巨大,低概率项→0 | 只挑最稳的,**千篇一律但可靠** | 只点你吃过一百遍的"安全菜",绝不踩雷 |
| **中**(0.3~0.7) | 自然分布 | 平衡准确与多样 | 正常状态,偶尔小冒险 |
| **高**(0.7~2) | 分布被"摊平",低概率词也被抬举上来 | 天马行空,**可能出彩也可能翻车** | 喝了两杯开始点菜单上从没试过的怪菜 |

> 📐 公式直觉:T 越大 → 指数被摊平 → 本来排第 50 名的词也突然有了不小概率。T→∞ → 完全随机;T→0 → 永远选第一名(贪婪)。

**Top-k——"只留前 k 名"**

把所有候选词按概率从高到低排,**只留前 k 个**,后面的直接淘汰,再在这 k 个里归一化后采样。

- k=1 → 贪婪:永远选第一名(最保守)
- k=50 → 给前 50 名各自按比例分配机会

**Top-p(核采样)——"够了就关门"**

不固定人数,而是从最高开始拿,攒到**累积概率 ≥ p** 为止,然后就在这几个里采样:

- p=0.9 → 前 3 个词就攒够 90% 概率 → 只在这 3 个里选(安全)
- p=0.95 → 需要前 20+ 个词才能攒到 95% → 选择面更广(更多样)

> Top-p 比 Top-k 聪明在 **"动态适应"**:概率集中的时候自动少给你选项(安全),分散的时候自动多给(发散)。

**三旋钮协同**:Temperature 重新分配财富 → Top-k 淘汰尾部 → Top-p 够了就关门。通常 **Top-k 和 Top-p 二选一就够**,否则海选 k 名后再关门到 p,交集更小。

#### 🧠 指令调优:把"续写机器"掰成"听命令的助手"

早期 GPT-3 是一个**文本续写器**。你跟它说"把下面翻成英文",它续写的是"…is often used in programming…"——它根本不知道你的意思是"你来帮我翻译"。因为它受的训练只有"接下去",没有"服从指令"。

**指令调优**:在预训练完之后,额外用大量"(指令→执行结果)"格式的数据继续训练,把这个续写器**掰成听人话的助手**。

> ChatGPT、DeepSeek、Qwen、Claude——你今天用的所有对话模型,全部经过这道工序。**这是 LLM 从"语言模型"升级为"智能体大脑"最关键的一跳。**

---

#### 💡 思维链 CoT:"把草稿写在纸上"

一个被反复验证的洞见:让模型直接报答案 → 容易错。但在 prompt 末尾加一句 **"请一步步思考"** → 准确率飙升。

**为什么有效?** Decoder-Only 模型生成每个词,都是基于**它前面已经生成的所有词**(自回归)。如果你让它先写:
```
"第一步:第一个赛季 80场×60%=48场胜利。
 第二步:总场数=80+15=95,总胜场=48+12=60。
 第三步:60÷95≈63.16%。"
```

这些中间步骤**成了后续计算可用的上下文**。等于让模型在纸上打草稿,减少脑内跳步算错。

> 🔑 **CoT 把"脑子里的暗箱推理"变成了"纸上的明账"**——而这份明账本身,也正是 LLM 作为神经符号系统的证明:神经网络做直觉跳跃,但把思考过程用符号(文字)摊开了。

### 3.2.2 文本分词:切菜机器人

计算机只认数字。把"今天天气真好"翻译成一串数字的过程 = **分词(Tokenization)**。分词器(Tokenizer)就是这台**"切菜机器人"**。

#### 三种切法

| 切法 | 怎么切 | 优点 | 致命缺陷 | 类比 |
|------|--------|------|---------|------|
| **按词切** | 空格/标点断句 | 直观,一个词=一个 token | "look"/"looks"/"looking"是三个互不相关的 token;遇到没见过的词→OOV(傻眼) | 按整棵白菜卖——白菜、娃娃菜、紫甘蓝各算各的,来了个你不认识的菜直接没法处理 |
| **按字符切** | 一个字一个字 | 词表小(几十个字符),永远没有 OOV | "agent"→["a","g","e","n","t"],五个字符没一个单独有意义,模型学得累 | 把每片叶子摘下来单独卖——都是叶子,你得自己拼出"这是白菜" |
| **子词 BPE** | 高频全保留,低频拆片段 | 词表可控+能处理新词+保留语义 | 需要训练分词器 | 白菜整颗卖;不认识的"抱子甘蓝"→拆成"抱子"+"甘蓝",你至少认识"甘蓝" |

#### BPE 算法 = 一条贪吃蛇

BPE(Byte-Pair Encoding)是 GPT 系列用的算法。训练过程:

```
初始:每个字符单独一个 token (h, u, g, p, n, b…)
第1轮:统计语料里所有相邻 token 对的出现频率 → "u"+"g"最常挨着 → 合并成 "ug"
第2轮:"ug"+"</w>"(词尾标记)最常挨着 → 合并成 "ug</w>"
第3轮:"u"+"n"最常挨着 → 合并成 "un"
第4轮:"un"+"</w>"最常挨着 → 合并成 "un</w>"
…
一直合并,直到词表大小达到预设值(比如 GPT-4 的词表是 10 万)。
```

**碰到生词怎么办?** 比如训练时没见过 "bug" → 试着匹配"bug"(没有)→ 拆成"b"+"ug"(有了!因为 ug 是高频子词)→ 输出 ['b','ug']。

#### 为什么切菜机器人对智能体开发者这么重要

| 原因 | 解释 |
|------|------|
| **上下文窗口按 Token 算,不是按字数** | 同样的一句话,**中文可能花掉比英文多 1.5~3 倍的 Token**(因为 BPE 是拿英文语料训练出来的,切中文天然钝)。你的 128K 上下文窗口,如果全喂中文,实际有效信息量可能只有英文的 1/3 |
| **API 按 Token 计费** | 你的智能体每调一次 LLM,账单都按 Token 数算。不分词你就不知道钱烧在哪 |
| **诡异 bug 的根因** | 模型算 `2+2` 很对,但算 `2+2`(没空格)可能错——因为后者被分词器当成了一个**不常见的独立 Token**,而不是"2"+"+"+"2"三个熟悉的 token |

### 3.2.3 调用开源大模型:把大脑搬到自己电脑上

这一节是实操,用 HuggingFace Transformers 加载 Qwen1.5-0.5B 跑本地推理。流程骨架:

```
加载模型 + 分词器 → 构造对话消息 → tokenize(文本→数字ID) → generate(模型推理) → decode(数字ID→文本)
```

**API vs 本地的抉择**(智能体开发者两种都要用):

| | 🌐 API 调用 | 🏠 本地部署 |
|---|---|---|
| **速度** | 快,不占自己显卡 | 慢,吃显卡/内存 |
| **成本** | 按 Token 计费,用得越多越贵 | 一次性硬件投入+电费 |
| **隐私** | 数据发到第三方 | 数据不离开你的机器 |
| **定制** | 有限(部分服务商支持微调) | 完全自由:微调、量化、改架构 |
| **适用场景** | 开发调试、快速原型 | 敏感数据、离线环境、高定制需求 |

---

### 3.2.4 模型选型:不是买最贵的车,是买对的那辆

书里列了 GPT/Gemini/Claude/Llama/Mistral/Qwen 等一堆模型名,名字会过时,**但选型逻辑不会过时**。

| 维度 | 你在选什么 | 🚗 买车类比 |
|------|-----------|-----------|
| **性能** | 它擅长啥?推理?代码?创意写作? | 跑车 vs SUV vs 货车——不同路况选不同车 |
| **成本** | 闭源按 Token 烧钱 / 开源吃显卡 | 打车(用一次付一次)vs 自己买车养车 |
| **速度(延迟)** | 实时交互(game NPC)不能等 3 秒 | 外卖(要快)vs 慢炖(等得起) |
| **上下文窗口** | 能一次性"记住"多少 | 草稿纸大小——解微积分需要 A4,写长篇小说需要卷筒纸 |
| **部署方式** | 数据给第三方 vs 留自己机房 | 住酒店(方便但没隐私)vs 买房(折腾但自由) |
| **生态工具链** | 踩坑时有多少人救你 | 选大众车→配件多、修理铺遍地;选冷门车→一个零件等三周 |
| **可微调性** | 能教它新本事吗 | 改装自由度——有的车 ECU 锁死,有的随便刷 |
| **安全伦理** | 偏见/幻觉/毒性控制水平 | 刹车和保险——开得再快,安全是底线 |

> 🎯 金句:**一个需要 24fps 实时互动的游戏 NPC,不能用 GPT-5 每次等 3 秒;一个分析机密法律合同的智能体,不能把数据发给第三方 API。选模型,不是选冠军,是选搭档。**

## 3.3 缩放法则与幻觉:引擎的物理定律与天生缺陷

### 3.3.1 缩放法则 = RPG 游戏的"升级曲线"

> **模型性能(用 Loss 衡量)与参数量、数据量、计算量之间,存在可预测的幂律关系。** 在对数-对数坐标上画出来是一条**漂亮的直线**——你按比例加资源,性能就按规律提升,不会突然撞墙。

**Chinchilla 定律:别光怼肌肉,智力也得点**

早期大家迷信"越大越好":GPT-3 有 1750 亿参数,像一个 2 米的壮汉但**只上过小学**。

2022 年 DeepMind 的 Chinchilla 模型说了一句打脸的话:

> **在固定计算预算下,最优方案是:模型比你以为的应该小一点,但训练数据量要远比你以为的多得多。**

| | GPT-3 | Chinchilla |
|---|---|---|
| 参数量(肌肉) | 1750 亿 💪💪💪 | 700 亿 💪 |
| 训练数据量(读书量) | ~3000 亿 token 📖 | ~1.4 万亿 token 📖📖📖📖📖 |
| 结果 | 基准线 | **性能反超** ✅ |

> 🎯 **读书量比肌肉量重要。** Llama 系列后来的成功,直接继承了 Chinchilla 定律的思路。就像一个博览群书的瘦子,比一个只读过三本书的壮汉更聪明。

**涌现能力:过了一条隐形的线,突然就会了**

研究发现:**当模型参数量跨越某个阈值后,会突然展现出小模型完全不会的全新能力。**

```
模型很小(几亿参数)  → 指令遵循?❌  多步推理?❌  CoT?❌  代码生成?😅
模型够大(几百亿+)   → 指令遵循?✅  多步推理?✅  CoT?✅  代码生成?✅
```

就像 RPG 游戏:**10 级时只会平砍。升到 20 级,突然自动解锁了必杀技**——没人专门教过这个技能,是"属性够了"自然激活的。

> ⚠️ **对智能体开发者的含义**:想用 CoT(思维链)、复杂规划(ReAct/Plan-Solve)这些高级能力,**底座模型得够大**。你用一个小模型去跑 ReAct 循环,它连"先想再动"这个纪律都守不住——不是范式不好,是引擎马力不够。

---

### 3.3.2 幻觉:天生脑补,不是 bug 是机制

#### 什么是幻觉(一个精准定义)

> 模型生成的内容,跟**客观事实**、用户输入的**上下文**或**自身已生成的内容**相矛盾。

换个说法:模型不是在"说话",它是在**续写故事**。它每步只做一件事——"接下来,什么词最可能?"。它没有内置事实核查器。

#### 三种幻觉(从轻到重)

| 类型 | 表现 | 🎭 即兴表演类比 |
|------|------|---------------|
| **事实性幻觉** Factual | 编造不存在的事实/实体 | 演员即兴说"我来自火星的'新北京城'"——故事里顺,但现实中没有 |
| **忠实性幻觉** Faithfulness | 摘要/翻译偏离原文意思 | 剧本写"他恨她",演员总结成"他们相爱" |
| **内在幻觉** Intrinsic | 自己打自己脸 | 上句说"A > B",下句说"B > A" |

#### 根因:自回归 = 一直在即兴表演,没有台词本

> LLM 每步产出都基于"训练时见过的统计模式 × 到目前为止已经生成的所有词"。这个机制确保**接得顺**(符合语言习惯),但不保证**接得对**(符合事实)。

就像让你即兴讲一个"古罗马历史":如果你读过很多罗马史 → 能讲出凯撒、元老院、高卢战争 → 但讲到一半发现缺了一块,你的大脑会自动"脑补"一个听起来合理但实际上不存在的细节。**你不是在撒谎,你是在即兴接龙,而大脑的接龙引擎不区分"事实"和"合理的虚构"。**

#### 三层缓解策略(= 后面章节的地图)

| 层级 | 怎么缓解 | 书里哪章展开 |
|------|---------|------------|
| **数据层**(源头) | 清洗训练数据 + RLHF(人类反馈强化学习) | 第十一章 Agentic RL |
| **推理层**(使用时) | ① **RAG(检索增强生成)**:生成前先查外部知识库,拿着"标准答案"再写 ② 多步推理+自我验证 ③ 调外部工具(搜索引擎/计算器/代码解释器) | **第八章 RAG**、第九章 上下文工程 |
| **模型层**(架构) | 探索新架构、让模型能输出"我不确定" | 研究前沿 |

> 🎯 **对 Agent 架构最重要的教训**:**别把 LLM 当维基百科用,把它当"推理引擎 + 语言界面"用。**
>
> - LLM 的活儿:**理解**你的问题、**规划**怎么查、**把查到的东西揉成**人话
> - 不该 LLM 干的:**存储事实**、**精确计算**、**实时信息**——这些丢给 RAG、计算器、搜索引擎
>
> 这是整个 Agent 架构设计的**根本分界线**。幻觉不会被消灭,只能被管理。好的 Agent 设计,就是给这个"即兴表演天才"配一个靠谱的"提词器 + 事实核查员"。
