## 16.2 项目选题指南

### 16.2.1 选题原则

**SMART原则：**

- **S**pecific（具体）：明确要解决什么问题
- **M**easurable（可衡量）：有清晰的成功标准
- **A**chievable（可实现）：1-2周内能完成
- **R**elevant（相关）：与课程内容相关
- **T**ime-bound（有时限）：设定截止日期

**好选题的特征：**
- ✅ 解决实际问题
- ✅ 技术难度适中
- ✅ 有创新点
- ✅ 可展示性强

**避免的选题：**
- ❌ 过于简单（如简单聊天机器人）
- ❌ 过于复杂（如AGI系统）
- ❌ 缺乏创新（完全复制现有项目）

### 16.4.4 测试你的项目

**单元测试：**

```python
# tests/test_agent.py
import pytest
from my_project.agent import MyAgent

def test_agent_creation():
    """测试Agent创建"""
    agent = MyAgent(name="测试")
    assert agent.name == "测试"

def test_agent_response():
    """测试Agent响应"""
    agent = MyAgent()
    result = agent.run("你好")
    assert result is not None
    assert len(result) > 0

@pytest.mark.asyncio
async def test_async_agent():
    """测试异步Agent"""
    agent = MyAgent()
    result = await agent.async_run("测试")
    assert result is not None
```

**运行测试：**
```bash
pytest tests/ -v
```

### 16.5.1 提交代码到 GitHub

```bash
# 1. 创建新分支

# 2. 添加文件

# 3. 提交
git commit -m "Add: 我的项目名称

# 4. 推送到GitHub
git push origin feature/my-project
```

### 16.5.2 创建 Pull Request

**步骤：**

1. 访问你的GitHub仓库页面
2. 点击 "Compare & pull request"
3. 填写PR标题和描述

**PR模板：**
```markdown

- [x] 应用项目

## 项目简介

- 特性2
- 特性3

## 演示
[演示视频链接或截图]

## 测试
- [x] 已通过所有单元测试
- [x] 已在本地完整运行
```

### 16.5.3 响应 Review 意见

**常见Review意见：**

1. **代码规范问题**

2. **文档问题**
   - 补充使用说明
   - 添加架构图
   - 完善示例

3. **功能问题**
   - 修复bug
   - 优化性能
   - 增强容错

**回复示例：**
