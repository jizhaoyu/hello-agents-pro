### 详细设计

#### 1. Master Agent（主控）

**职责：**
- 解析用户请求
- 路由到专业Agent
- 汇总结果
- 生成最终回复

**实现：**
```python
class MasterAgent:
    def __init__(self):
        self.planner = PlannerAgent()    # 计划制定
        self.tutor = TutorAgent()        # 知识讲解
        self.examiner = ExaminerAgent()  # 出题评测
    
    def process(self, user_input: str) -> str:
        # 1. 理解用户意图
        intent = self.classify_intent(user_input)
        
        # 2. 路由到对应Agent
        if intent == "plan":
            return self.planner.create_plan(user_input)
        elif intent == "explain":
            return self.tutor.explain(user_input)
        elif intent == "practice":
            return self.examiner.generate_quiz(user_input)
        
        # 3. 默认综合处理
        return self.comprehensive_help(user_input)
```

#### 2. Planner Agent（计划制定）

**职责：**
- 分析学习目标

**工作流程：**

**特性：**

### 功能2：知识点深度讲解

**示例：讲解"Python装饰器"**

**1. 概念介绍**
```
装饰器是Python的一个强大特性，本质上是一个函数，
用于包装另一个函数，在不修改原函数代码的情况下，
增加额外的功能。
```

**2. 生活类比**
```
想象你有一杯咖啡（原函数），装饰器就像是：
- 加糖（增加功能A）
- 加奶（增加功能B）
- 加冰（增加功能C）
咖啡本身不变，但体验变丰富了。
```

**3. 代码示例**
```python
# 简单装饰器
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"调用函数：{func.__name__}")
        result = func(*args, **kwargs)
        print(f"返回结果：{result}")
        return result
    return wrapper

@logger
def add(a, b):
    return a + b

add(3, 5)
# 输出：
# 调用函数：add
# 返回结果：8
```

**4. 应用场景**
- 日志记录

- 权限验证
- 缓存处理

**5. 互动练习**

```
1. 记录函数开始时间
2. 执行函数

4. 打印执行耗时
```

```
装饰器的本质是什么？
A. 一个类

C. 一个模块
D. 一个变量

答案：B
```

**题目2（代码理解）**
```python
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def greet(name):
    print(f"Hello, {name}!")

greet("Alice")

# 这段代码会输出什么？
# A. Hello, Alice!
# B. Hello, Alice! Hello, Alice! Hello, Alice!
# C. 报错
# D. 什么都不输出

答案：B
解析：repeat装饰器会让函数执行3次
```

## 7.5 性能优化

### 1. 响应速度优化

**问题：** LLM调用延迟高（2-5秒）

**解决方案：**

**a) 流式输出**
```python
def stream_response(prompt):
    for chunk in llm.stream(prompt):
        yield chunk  # 立即返回部分结果
        
# 用户可以立即看到开始输出
```

**b) 缓存常见问题**
```python
cache = {}

def get_answer(question):
    if question in cache:
        return cache[question]  # 命中缓存，立即返回
    
    answer = llm.generate(question)
    cache[question] = answer
    return answer
```

**c) 预加载**
```python
# 应用启动时预生成常见内容
common_questions = [
    "什么是Python？",
    "如何学习编程？",
    ...
]

for q in common_questions:
    cache[q] = llm.generate(q)
```

### 2. 成本优化

**问题：** LLM API调用费用高

**解决方案：**

**a) 智能路由**
```python
def route_request(query):
    complexity = assess_complexity(query)
    
    if complexity == "simple":
        return use_cheap_model(query)  # 使用便宜模型
    else:
        return use_powerful_model(query)  # 使用强大模型
```

**b) Prompt优化**
```python
# 优化前：冗长的prompt（1000 tokens）
# 优化后：精简的prompt（200 tokens）
# 节省80%的token费用
```

**c) 批量处理**
```python
# 合并多个请求为一个
requests = [req1, req2, req3]
combined_prompt = "\n".join(requests)
response = llm.generate(combined_prompt)
```

### 3. 并发性能

**问题：** 多用户同时访问，响应变慢

**解决方案：**

**a) 异步处理**
```python
import asyncio

async def handle_request(user_id, query):
    result = await llm.generate_async(query)
    return result

# 可以同时处理100+请求
```

**b) 队列管理**
```python
from queue import Queue

request_queue = Queue(maxsize=1000)
worker_pool = [Worker() for _ in range(10)]

# 10个worker并行处理请求
```

**c) 限流**
```python
from ratelimit import limits

@limits(calls=100, period=60)  # 每分钟100次
def api_call(query):
    return llm.generate(query)
```

```python
def safe_llm_call(prompt, max_retries=3):
    for i in range(max_retries):

            return llm.generate(prompt)
        except TimeoutError:
```
