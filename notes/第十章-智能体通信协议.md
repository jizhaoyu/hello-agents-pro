## 10.1 智能体通信协议基础

### 10.1.1 为何需要通信协议

**问题场景：**
```python
# 没有协议时的混乱
agent1.call_tool("search", query="天气")  # Agent1的工具调用方式
agent2.invoke("search", input="天气")     # Agent2的调用方式不同
agent3.use_function("search", q="天气")   # Agent3又是另一种方式

# 每个智能体实现都不同，无法互操作！
```

**有协议后：**
```python
# 使用标准MCP协议
mcp_client.call_tool(
    name="search",
    arguments={"query": "天气"}
)

# 所有遵循MCP的服务都用同样的方式调用
```

**协议的价值：**
- **互操作性** - 不同智能体可以无缝协作
- **可复用性** - 工具/服务可以被任何智能体使用
- **标准化** - 统一的接口降低学习成本
- **生态系统** - 促进社区贡献和工具共享

### 10.1.2 三种协议设计理念比较

| 协议 | 定位 | 通信模式 | 典型场景 |
|------|------|----------|----------|
| **MCP** | LLM访问外部工具/数据 | 客户端-服务器 | 智能体调用天气API、数据库查询 |
| **A2A** | 智能体间直接通信 | 点对点 | 两个智能体协作完成任务 |
| **ANP** | 智能体网络发现 | 服务注册/发现 | 在网络中找到合适的智能体 |

**类比理解：**
```
MCP ≈ HTTP/REST API
    智能体像浏览器，通过标准协议访问各种Web服务

A2A ≈ WebSocket
    两个智能体建立持久连接，实时双向通信

ANP ≈ DNS + 服务注册中心
    在网络中发现"谁能提供什么服务"
```

## 10.2 MCP 协议实战

### 10.2.1 MCP 协议概念介绍

**MCP（Model Context Protocol）** 是由Anthropic提出的协议，让LLM能够安全、标准化地访问外部工具和数据源。

**核心概念：**

```
┌─────────────────────────────────────────┐
│         MCP 客户端（Client）              │
│      (智能体/LLM应用)                     │
│                                         │
│  ┌────────┐  ┌────────┐  ┌────────┐   │
│  │ Tool1  │  │ Tool2  │  │ Tool3  │   │
│  └────────┘  └────────┘  └────────┘   │
└──────────────┬──────────────────────────┘
               │ MCP Protocol
               │ (JSON-RPC 2.0)
               │
┌──────────────┴──────────────────────────┐
│         MCP 服务器（Server）              │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │  Resources (资源)                │   │
│  │  - 文档、数据、配置等             │   │
│  └─────────────────────────────────┘   │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │  Tools (工具)                    │   │
│  │  - 可执行的函数                  │   │
│  └─────────────────────────────────┘   │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │  Prompts (提示词模板)            │   │
│  │  - 预定义的提示词                │   │
│  └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

**MCP的三大核心能力：**

1. **Resources（资源）**
   - 智能体可以读取的数据源
   - 例如：文档、配置文件、数据库记录

2. **Tools（工具）**
   - 智能体可以调用的函数
   - 例如：搜索、计算、API调用

3. **Prompts（提示词）**
   - 预定义的提示词模板
   - 例如：代码审查模板、文档生成模板

### 10.2.2 使用 MCP 客户端

**安装：**
```bash
pip install mcp
```

**基本使用：**
```python
from mcp import Client, StdioServerParameters
from mcp.client.stdio import stdio_client

# 1. 连接到MCP服务器
server_params = StdioServerParameters(
    command="python",
    args=["weather_server.py"]
)

async with stdio_client(server_params) as (read, write):
    async with Client(read, write) as client:
        # 2. 初始化连接
        await client.initialize()
        
        # 3. 列出可用工具
        tools = await client.list_tools()
        print("可用工具:", [t.name for t in tools])
        
        # 4. 调用工具
        result = await client.call_tool(
            name="get_weather",
            arguments={"city": "北京"}
        )
        print(result)
```

### 10.2.3 在智能体中使用 MCP 工具

**集成到 HelloAgents：**
```python
from hello_agents import SimpleAgent, HelloAgentsLLM
from hello_agents.tools.mcp_tool import MCPTool

# 创建MCP工具
mcp_tool = MCPTool(
    server_command="python",
    server_args=["weather_server.py"]
)

# 创建智能体并注册工具
agent = SimpleAgent(
    name="天气助手",
    llm=HelloAgentsLLM(),
    tools=[mcp_tool]
)

# 使用智能体
response = agent.run("北京今天天气怎么样？")
print(response)
```

**工作流程：**
```
1. 用户问题："北京今天天气怎么样？"
   ↓
2. Agent推理：需要调用天气查询工具
   ↓
3. MCPTool.call_tool("get_weather", {"city": "北京"})
   ↓
4. MCP服务器执行查询，返回结果
   ↓
5. Agent基于结果生成自然语言回复
```

## 10.3 A2A 协议实战

### 10.3.1 协议设计动机

**A2A（Agent-to-Agent）** 协议用于智能体之间的直接通信。

**设计目标：**
- 智能体可以互相请求服务
- 支持异步消息传递
- 保持会话状态

**消息格式：**
```json
{
  "protocol": "A2A/1.0",
  "message_id": "msg_123",
  "from": "agent_researcher",
  "to": "agent_writer",
  "type": "request",
  "action": "write_article",
  "payload": {
    "topic": "AI智能体发展史",
    "length": 2000,
    "style": "学术"
  },
  "context": {
    "conversation_id": "conv_456",
    "previous_messages": []
  },
  "timestamp": "2024-01-15T10:30:00Z"
}
```

### 10.3.2 使用 A2A 协议实战

**场景：研究员Agent请求写作Agent撰写文章**

```python
from hello_agents.protocols.a2a import A2AClient, A2AServer

# Agent 1: 研究员（发送请求）
class ResearcherAgent:
    def __init__(self):
        self.a2a_client = A2AClient()
    
    async def request_article(self, topic):
        # 发送A2A消息
        response = await self.a2a_client.send_message(
            to="agent_writer",
            action="write_article",
            payload={
                "topic": topic,
                "length": 2000,
                "style": "学术"
            }
        )
        return response

# Agent 2: 写作者（接收请求并处理）
class WriterAgent:
    def __init__(self):
        self.a2a_server = A2AServer(agent_id="agent_writer")
        self.a2a_server.register_handler("write_article", self.handle_write)
    
    async def handle_write(self, message):
        topic = message["payload"]["topic"]
        length = message["payload"]["length"]
        
        # 执行写作任务
        article = self.write_article(topic, length)
        
        # 返回响应
        return {
            "status": "success",
            "result": article
        }
    
    def write_article(self, topic, length):
        # 调用LLM生成文章
        prompt = f"请撰写一篇关于{topic}的文章，长度约{length}字"
        return self.llm.generate(prompt)

# 使用示例
researcher = ResearcherAgent()
writer = WriterAgent()

# 研究员请求写作
result = await researcher.request_article("智能体通信协议")
print(result)
```

## 10.4 ANP 协议实战

### 10.4.1 协议目标

**ANP（Agent Network Protocol）** 是服务发现协议，帮助智能体在网络中找到合适的协作者。

**类比：** 就像你在一个大会场找专家咨询
- 不需要知道每个专家的具体位置
- 只需要说"我需要一个AI专家"
- 系统自动帮你找到并连接

### 10.4.2 使用 ANP 服务发现

**注册服务：**
```python
from hello_agents.protocols.anp import ANPRegistry

# 创建注册中心
registry = ANPRegistry()

# Agent注册自己的服务
registry.register(
    agent_id="agent_weather",
    capabilities=["天气查询", "气象分析"],
    endpoint="http://localhost:8001",
    metadata={
        "version": "1.0",
        "supported_cities": ["北京", "上海", "深圳"]
    }
)

registry.register(
    agent_id="agent_translator",
    capabilities=["翻译", "多语言"],
    endpoint="http://localhost:8002"
)
```

**发现服务：**
```python
# 查找能提供"天气查询"服务的智能体
weather_agents = registry.discover(capability="天气查询")

for agent in weather_agents:
    print(f"找到智能体: {agent['agent_id']}")
    print(f"端点: {agent['endpoint']}")
    
    # 调用该智能体
    response = requests.post(
        agent['endpoint'],
        json={"action": "get_weather", "city": "北京"}

    print(response.json())
```

### 10.4.3 实战案例：智能任务分发

```python
class TaskCoordinator:
    """任务协调者：自动发现并分配任务给合适的智能体"""
    
    def __init__(self, registry):
        self.registry = registry
    
    async def execute_task(self, task_description):
        # 1. 分析任务需要什么能力
        required_capabilities = self.analyze_task(task_description)
        
        # 2. 发现具备这些能力的智能体
        suitable_agents = []
        for cap in required_capabilities:
            agents = self.registry.discover(capability=cap)
            suitable_agents.extend(agents)
        
        # 3. 分配任务
        results = []
        for agent in suitable_agents:
            result = await self.call_agent(agent, task_description)
            results.append(result)
        
        # 4. 整合结果
        return self.merge_results(results)
    
    def analyze_task(self, task):
        """分析任务需要哪些能力"""
        # 使用LLM分析
        prompt = f"""
        分析以下任务需要哪些智能体能力：
        {task}
        
        请列出需要的能力（如：搜索、翻译、数据分析等）
        """
        capabilities = self.llm.generate(prompt)
        return capabilities.split(",")

# 使用
coordinator = TaskCoordinator(registry)
result = await coordinator.execute_task(
    "查询北京天气，并翻译成英文"
)
```

## 10.5 构建自定义 MCP 服务器

### 10.5.1 创建你的第一个 MCP 服务器

**场景：创建一个计算器MCP服务**

```python
from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import Tool, TextContent

# 创建服务器
app = Server("calculator-server")

# 定义工具
@app.list_tools()
async def list_tools() -> list[Tool]:
    return [
        Tool(
            name="add",
            description="两数相加",
            inputSchema={
                "type": "object",
                "properties": {
                    "a": {"type": "number", "description": "第一个数"},
                    "b": {"type": "number", "description": "第二个数"}
                },
                "required": ["a", "b"]
            }
        ),
        Tool(
            name="multiply",
            description="两数相乘",
            inputSchema={
                "type": "object",
                "properties": {
                    "a": {"type": "number"},
                    "b": {"type": "number"}
                },
                "required": ["a", "b"]
            }
        )
    ]

# 实现工具逻辑
@app.call_tool()
async def call_tool(name: str, arguments: dict):
    if name == "add":
        result = arguments["a"] + arguments["b"]
        return [TextContent(
            type="text",
            text=f"结果: {result}"
        )]
    
    elif name == "multiply":
        result = arguments["a"] * arguments["b"]
        return [TextContent(
            type="text",
            text=f"结果: {result}"
        )]
    
    else:
        raise ValueError(f"未知工具: {name}")

# 运行服务器
if __name__ == "__main__":
    import asyncio
    asyncio.run(stdio_server(app))
```

**使用自定义服务器：**
```python
# 客户端代码
from mcp import Client, StdioServerParameters
from mcp.client.stdio import stdio_client

async def use_calculator():
    server_params = StdioServerParameters(
        command="python",
        args=["calculator_server.py"]
    )
    
    async with stdio_client(server_params) as (read, write):
        async with Client(read, write) as client:
            await client.initialize()
            
            # 调用加法
            result = await client.call_tool(
                name="add",
                arguments={"a": 10, "b": 20}
            )
            print(result)  # 结果: 30
            
            # 调用乘法
            result = await client.call_tool(
                name="multiply",
                arguments={"a": 5, "b": 6}
            )
            print(result)  # 结果: 30

asyncio.run(use_calculator())
```

## 10.6 本章总结

### 核心要点

1. **通信协议是多智能体系统的基础**
   - MCP: LLM访问外部工具/数据
   - A2A: 智能体间直接通信
   - ANP: 服务发现与注册

2. **MCP协议最成熟**
   - Anthropic官方支持
   - 丰富的社区生态
   - 标准化的工具/资源/提示词

3. **构建自定义MCP服务很简单**
   - 定义工具schema
   - 实现工具逻辑
   - 注册到服务器

### 最佳实践

- ✅ 优先使用MCP协议集成外部工具
- ✅ 为工具提供清晰的描述和schema
- ✅ 使用ANP实现动态服务发现
- ✅ A2A协议适合智能体间复杂交互
- ✅ 遵循协议规范，提升互操作性
