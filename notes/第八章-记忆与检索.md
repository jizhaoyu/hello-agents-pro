## 8.1 从认知科学到智能体记忆

### 8.1.1 人类记忆系统的启发

**人类记忆的三层结构：**

```
感觉记忆 (Sensory Memory)
    ↓ 注意力筛选
短期记忆 (Short-term Memory) 
    ↓ 编码 & 复述
长期记忆 (Long-term Memory)
```

**关键特征：**
- **容量限制** - 短期记忆只能保持7±2个项目（米勒定律）
- **遗忘曲线** - 艾宾浩斯发现记忆随时间衰减
- **情景记忆** - 特定时间地点的个人经历
- **语义记忆** - 抽象的事实和概念知识

### 8.1.2 为何智能体需要记忆与RAG

**LLM的局限性：**

1. **上下文窗口限制** - 即使是GPT-4，也只能处理有限tokens
2. **知识截止日期** - 训练数据有时效性
3. **无个性化记忆** - 每次对话都是全新开始
4. **幻觉问题** - 可能编造不存在的信息

**记忆系统的价值：**
- 维持长期对话的连贯性
- 个性化用户体验
- 跨会话的信息保持

**RAG系统的价值：**
- 访问最新的外部知识
- 引用可验证的信息源
- 减少幻觉和错误
- 处理私有/专业领域知识

### 8.2.1 记忆系统的工作流程

**完整生命周期：**

```python
# 1. 存储记忆
memory.store({

# 2. 检索记忆
relevant_memories = memory.retrieve(

# 4. 生成回复
```

### 8.2.2 四种记忆类型

#### 1. 短期记忆 (Short-term Memory)

**特点：**
- 存储最近的对话历史
- 容量有限（通常最近10-20轮）
- 随对话结束而清空

**实现：**
```python
class ShortTermMemory:
    def __init__(self, max_turns=10):
        self.messages = []
        self.max_turns = max_turns
    
    def add(self, role, content):
        self.messages.append({
            "role": role,
            "content": content,
            "timestamp": datetime.now()
        })
        
        # 超出限制则删除最早的
        if len(self.messages) > self.max_turns * 2:
            self.messages = self.messages[-self.max_turns * 2:]
    
    def get_context(self):
        return self.messages
```

#### 2. 长期记忆 (Long-term Memory)

**特点：**
- 永久保存重要信息
- 使用向量数据库存储
- 通过语义相似度检索

**实现：**
```python
class LongTermMemory:
    def __init__(self, embedding_model, vector_db):
        self.embedding_model = embedding_model
        self.vector_db = vector_db
    
    def store(self, content, metadata=None):
        # 1. 生成向量
        embedding = self.embedding_model.encode(content)
        
        # 2. 存储到向量数据库
        self.vector_db.add(
            vector=embedding,
            content=content,
            metadata=metadata
        )
    
    def retrieve(self, query, top_k=5):
        # 1. 查询向量化
        query_embedding = self.embedding_model.encode(query)
        
        # 2. 相似度检索
        results = self.vector_db.search(
            query_embedding,
            top_k=top_k
        )
        
        return results
```

#### 3. 工作记忆 (Working Memory)

**特点：**
- 存储当前任务的中间结果
- 类似于人脑的"工作台"
- 任务完成后可选择性保留

**应用场景：**
```python
# 多步推理任务
working_memory = {
    "task": "分析市场趋势",
    "step1_result": "收集了2023年数据",
    "step2_result": "发现增长率为15%",
    "current_step": 3,
    "intermediate_thoughts": [...]
}
```

#### 4. 情景记忆 (Episodic Memory)

**特点：**
- 记录完整的交互片段
- 包含时间、地点、上下文
- 用于回顾和学习

**数据结构：**
```python
episode = {
    "id": "ep_20240115_001",
    "start_time": "2024-01-15 10:00:00",
    "end_time": "2024-01-15 10:15:00",
    "user_id": "user_123",
    "conversation": [...],  # 完整对话
    "task_completed": True,
    "satisfaction_score": 4.5,
    "key_events": [
        "用户提出问题",
        "智能体搜索信息",
        "提供解决方案",
        "用户表示满意"
    ]
}
```

## 8.3 RAG系统：知识检索增强

### 8.3.1 RAG的基础知识

**RAG (Retrieval-Augmented Generation) 的核心思想：**

在生成回答之前，先从外部知识库检索相关信息，然后基于检索到的内容生成答案。

**RAG vs 微调：**

| 特性 | RAG | 微调 |
|------|-----|------|
| **知识更新** | 实时更新知识库即可 | 需要重新训练模型 |
| **成本** | 低（只需存储文档） | 高（需要GPU训练） |
| **可解释性** | 高（可追溯信息源） | 低（知识编码在参数中） |
| **准确性** | 基于事实文档 | 可能产生幻觉 |
| **适用场景** | 知识密集型任务 | 任务适配、风格调整 |

### 8.3.2 RAG系统工作原理

**完整流程：**

```
1. 离线索引阶段
   文档 → 分块 → 向量化 → 存入向量数据库

2. 在线检索阶段
   用户问题 → 向量化 → 相似度检索 → 获取Top-K文档

3. 生成阶段
   问题 + 检索文档 → LLM → 生成答案
```

**详细步骤：**

```python
# 步骤1: 文档处理
def process_documents(documents):
    chunks = []
    for doc in documents:
        # 文档分块
        doc_chunks = split_text(
            doc.content,
            chunk_size=512,
            overlap=50
        )
        chunks.extend(doc_chunks)
    return chunks

# 步骤2: 向量化存储
def build_index(chunks, embedding_model):
    vectors = []
    for chunk in chunks:
        vector = embedding_model.encode(chunk)
        vectors.append(vector)
    
    # 存入向量数据库
    vector_db.add_batch(vectors, chunks)

# 步骤3: 检索
def retrieve(query, top_k=3):
    query_vector = embedding_model.encode(query)
    results = vector_db.search(query_vector, top_k)
    return results

# 步骤4: 生成答案
def generate_answer(question, context_docs):
    prompt = f"""
    参考以下文档回答问题：
    
    {context_docs}
    
    问题：{question}
    
    请基于上述文档内容回答，如果文档中没有相关信息，请说明。
    """
    
    answer = llm.generate(prompt)
    return answer
```

### 8.3.3 文档分块策略

**为什么需要分块？**
- 向量模型有输入长度限制
- 细粒度检索更精准
- 减少无关信息干扰

**常见分块方法：**

1. **固定长度分块**
```python
def fixed_length_split(text, chunk_size=512, overlap=50):
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunk = text[start:end]
        chunks.append(chunk)
        start = end - overlap  # 重叠部分
    return chunks
```

2. **语义分块（按段落）**
```python
def semantic_split(text):
    # 按段落分割
    paragraphs = text.split('\n\n')
    
    # 合并过短的段落
    chunks = []
    current_chunk = ""
    for para in paragraphs:
        if len(current_chunk) + len(para) < 512:
            current_chunk += para + "\n\n"
        else:
            chunks.append(current_chunk)
            current_chunk = para
    
    if current_chunk:
        chunks.append(current_chunk)
    
    return chunks
```

3. **递归分块**
```python
def recursive_split(text, max_size=512):
    if len(text) <= max_size:
        return [text]
    
    # 尝试按分隔符分割
    for separator in ['\n\n', '\n', '. ', ' ']:
        if separator in text:
            mid = len(text) // 2
            # 找最近的分隔符
            split_pos = text.find(separator, mid)
            if split_pos != -1:
                left = text[:split_pos]
                right = text[split_pos + len(separator):]
                return recursive_split(left, max_size) + \
                       recursive_split(right, max_size)
    
    # 实在找不到分隔符就强制切分
    return [text[:max_size], text[max_size:]]
```

### 8.3.4 高级检索策略

#### 1. 混合检索 (Hybrid Search)

结合关键词检索和向量检索：

```python
def hybrid_search(query, top_k=5, alpha=0.5):
    # 向量检索
    vector_results = vector_search(query, top_k * 2)
    
    # 关键词检索 (BM25)
    keyword_results = bm25_search(query, top_k * 2)
    
    # 结果融合
    combined_scores = {}
    for doc_id, score in vector_results:
        combined_scores[doc_id] = alpha * score
    
    for doc_id, score in keyword_results:
        combined_scores[doc_id] += (1 - alpha) * score
    
    # 排序取Top-K
    sorted_results = sorted(
        combined_scores.items(),
        key=lambda x: x[1],
        reverse=True
    )[:top_k]
    
    return sorted_results
```

#### 2. 重排序 (Reranking)

使用更强大的模型对初步检索结果重新排序：

```python
def rerank(query, candidate_docs, reranker_model):
    scores = []
    for doc in candidate_docs:
        # 计算query和doc的相关性分数
        score = reranker_model.score(query, doc)
        scores.append(score)
    
    # 根据新分数排序
    ranked_docs = [
        doc for _, doc in sorted(
            zip(scores, candidate_docs),
            reverse=True
        )
    ]
    
    return ranked_docs
```

#### 3. 查询扩展 (Query Expansion)

使用LLM改写查询以提高召回率：

```python
def expand_query(original_query, llm):
    prompt = f"""
    原始问题：{original_query}
    
    请生成3个语义相似但表达不同的问题变体：
    1.
    2.
    3.
    """
    
    expanded_queries = llm.generate(prompt)
    
    # 对所有查询进行检索
    all_results = []
    for query in [original_query] + expanded_queries:
        results = vector_search(query, top_k=3)
        all_results.extend(results)
    
    # 去重并返回
    return deduplicate(all_results)
```

## 8.4 构建智能文档问答助手

### 实战案例：PDF文档问答系统

```python
class DocumentQAAssistant:
    def __init__(self, llm, embedding_model):
        self.llm = llm
        self.embedding_model = embedding_model
        self.vector_db = VectorDatabase()
        self.memory = MemoryManager()
    
    def ingest_document(self, pdf_path):
        """索引文档"""
        # 1. 提取文本
        text = extract_text_from_pdf(pdf_path)
        
        # 2. 分块
        chunks = semantic_split(text, chunk_size=512)
        
        # 3. 向量化
        for i, chunk in enumerate(chunks):
            embedding = self.embedding_model.encode(chunk)
            self.vector_db.add(
                id=f"{pdf_path}_{i}",
                vector=embedding,
                content=chunk,
                metadata={"source": pdf_path}
            )
        
        print(f"已索引 {len(chunks)} 个文本块")
    
    def answer_question(self, question):
        """回答问题"""
        # 1. 检索相关文档
        relevant_docs = self.vector_db.search(
            self.embedding_model.encode(question),
            top_k=3
        )
        
        # 2. 构建提示词
        context = "\n\n".join([
            f"文档{i+1}：{doc['content']}"
            for i, doc in enumerate(relevant_docs)
        ])
        
        prompt = f"""
        基于以下文档内容回答问题：
        
        {context}
        
        问题：{question}
        
        要求：
        1. 仅基于提供的文档内容回答
        2. 如果文档中没有相关信息，请明确说明
        3. 回答要准确、简洁
        """
        
        # 3. 生成答案
        answer = self.llm.generate(prompt)
        
        # 4. 存储到记忆
        self.memory.store({
            "question": question,
            "answer": answer,
            "sources": [doc['metadata'] for doc in relevant_docs]
        })
        
        return {
            "answer": answer,
            "sources": relevant_docs
        }

# 使用示例
assistant = DocumentQAAssistant(llm, embedding_model)

# 导入文档
assistant.ingest_document("agents_book.pdf")

# 提问
result = assistant.answer_question("什么是ReAct范式？")
print(result["answer"])
print("信息来源:", result["sources"])
```

## 8.5 本章总结

### 核心要点

1. **记忆系统** - 智能体需要记忆来维持长期对话连贯性
   - 短期记忆：最近对话历史
   - 长期记忆：永久保存的重要信息
   - 工作记忆：当前任务中间结果
   - 情景记忆：完整的交互片段

2. **RAG系统** - 通过检索外部知识增强生成能力
   - 文档分块：固定长度、语义分块、递归分块
   - 向量检索：基于embedding的相似度搜索
   - 高级策略：混合检索、重排序、查询扩展

3. **实践经验**
   - 合理的分块大小（通常512-1024 tokens）
   - 适当的重叠（50-100 tokens）
   - 结合向量检索和关键词检索
   - 记录信息来源以提高可信度

### 实践建议

- ✅ 根据应用场景选择合适的记忆类型
- ✅ RAG系统的文档质量直接影响答案质量
- ✅ 定期清理和更新知识库
- ✅ 监控检索的准确率和召回率
- ✅ 为检索结果添加来源引用
